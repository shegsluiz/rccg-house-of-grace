## 2024-05-11 - Adding ARIA labels and focus states to mobile menu and social links
**Learning:** Found that the app uses an overlay mobile menu and an intricate footer layout without sufficient ARIA labels on icon-only interactive elements. The text links embedded in navigation were lacking security-focused attributes `rel="noopener noreferrer"`. Incorporating standard utility classes (`focus-visible:ring-2 focus-visible:ring-hog-green-400 focus:outline-none`) integrates perfectly with the current project's theme logic (using `hog-green-400` from existing tokens) and greatly improves keyboard accessibility contextually without introducing bespoke CSS.
**Action:** When adding accessibility to interactive components, utilize the existing custom theme color tokens (like `hog-green-400`) within `focus-visible` ring utility classes to ensure a visually cohesive screen-reader and keyboard-friendly focus state. Consistently audit link targets for `noopener noreferrer` during such passes.

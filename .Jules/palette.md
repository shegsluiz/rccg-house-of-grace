## 2025-03-01 - Missing ARIA Labels on Icon-Only Elements and Form Inputs
**Learning:** Found a consistent pattern across components (`Layout.tsx`, `Home.tsx`) where icon-only interactive elements (mobile menu button, social links) and form inputs (newsletter email) were missing `aria-label`s. This makes the interface inaccessible to screen reader users, who would not know the purpose of these elements. Additionally, decorative SVGs were not explicitly hidden from screen readers using `aria-hidden="true"`, and explicit focus indicators were absent, making keyboard navigation difficult.
**Action:** Always ensure that icon-only buttons/links and form inputs without visible text labels include a descriptive `aria-label`. Hide purely decorative icons from screen readers using `aria-hidden="true"`, and implement visible focus states (`focus-visible:ring-2 focus-visible:ring-hog-green-400 focus:outline-none`) to improve keyboard accessibility.
